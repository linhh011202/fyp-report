\section{Overview}
\label{sec:auth_overview}

The Gateway Dashboard implements a comprehensive authentication system supporting multiple identity providers with robust session management. This section provides an overview of the authentication architecture.

\subsection{Supported Authentication Methods}

The system supports three authentication methods:

\begin{table}[H]
\centering
\begin{tabular}{|l|l|p{6cm}|}
\hline
\textbf{Method} & \textbf{Provider} & \textbf{Description} \\
\hline
Email/Password & Internal & Traditional credential-based login with bcrypt password hashing \\
\hline
Google OAuth 2.0 & Google & Social login using Passport's Google strategy \\
\hline
Apple Sign-In & Apple & Authentication via identity token verification \\
\hline
\end{tabular}
\caption{Supported Authentication Methods}
\label{tab:auth_methods}
\end{table}

\subsection{Module Architecture}

Figure~\ref{fig:sso_architecture} illustrates the overall architecture of the authentication modules.

\begin{figure}[H]
\centering
\includegraphics[width=0.75\textwidth]{images/sso_architecture.png}
\caption{Authentication Module Architecture}
\label{fig:sso_architecture}
\end{figure}

The architecture consists of three main components:

\begin{itemize}
    \item \textbf{SSO Modules}: GoogleSSOController and AppleSSOController handle OAuth flows with their respective providers.
    \item \textbf{Auth Module}: Central AuthService provides shared functionality including:
    \begin{itemize}
        \item \texttt{checkUserByEmail()}
        \item \texttt{generateTemporaryToken()}
        \item \texttt{revokeAllUserTokens()}
    \end{itemize}
    \item \textbf{Storage Layer}: UserTokens collection stores active sessions; TokenBlacklist stores revoked tokens; Users collection stores account data.
\end{itemize}

Both SSO controllers delegate user verification to the shared AuthService, ensuring consistent handling of new users (requiring password setup) and existing users (issuing access tokens).
